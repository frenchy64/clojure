# Synthetic Benchmark: Nil Optimization

This directory contains a focused synthetic benchmark to demonstrate the impact of the nil syntax-quote optimization on compiled bytecode.

## Overview

The benchmark compiles a Clojure namespace (`nil-benchmark.core`) that contains multiple macros heavily using syntax-quoted nil. It compares the bytecode generated by:

1. **Baseline**: Official Clojure 1.12.3 (from Maven Central)
2. **Optimized**: Current branch with nil optimization

## Test Cases

The `nil_benchmark_core.clj` file contains macros that exercise syntax-quoted nil in various scenarios:

- `when-not`: Simple conditional returning nil
- `defn-with-optional`: Default parameter as nil
- `safe-get`: Optional nil default in map access
- `when-let-or-nil`: Explicit nil return in bindings
- `defrecord-with-nil-defaults`: Multiple nil initializations

## Compilation Method

**Important**: This benchmark uses **direct `java -cp` commands** rather than Maven or Clojure CLI, as requested. This ensures:

- Minimal classpath (only Clojure JAR + source)
- Direct compilation with `-Dclojure.compiler.direct-linking=true`
- Full control over the compilation environment
- Reproducible builds

## Running the Benchmark

```bash
cd experiments/synthetic-benchmark-nil
./run-benchmark.sh
```

The script will:
1. Download official Clojure 1.12.3 JAR (with SHA256 verification)
2. Use the optimized Clojure JAR from the main experiment
3. Compile the test namespace with both versions using `java -cp`
4. Strip non-deterministic data (timestamps) from class files
5. Compare bytecode using `javap`
6. Generate detailed reports

## Output Files

Results are saved to `results/`:

- `summary.txt` - High-level comparison summary
- `*-baseline.txt` - Baseline bytecode (javap -c -v output)
- `*-optimized.txt` - Optimized bytecode (javap -c -v output)
- `*-diff.txt` - Side-by-side diff of bytecode changes

## What to Look For

If the nil optimization is effective, you should see:

1. **Fewer constant pool entries**: Less `CONSTANT_Methodref` for `quote` operations
2. **Simpler instruction sequences**: Direct `aconst_null` instead of quote form construction
3. **Smaller method bytecode**: Fewer instructions in compiled macro methods
4. **Changed class count**: All or most classes should show differences

## Integration with Main Experiment

This synthetic benchmark complements the main experiment by:

- **Isolating the effect**: Tests only nil-heavy code, not entire Clojure core
- **Demonstrating mechanism**: Shows exactly how bytecode changes
- **Providing evidence**: Clear before/after comparison for specific use cases
- **Validating hypothesis**: Confirms or refutes expected compilation changes

## Reproducibility

- Baseline JAR verified against known SHA256 checksum
- Uses direct Java commands (no build tools)
- Strip-nondeterminism ensures fair comparison
- All steps are automated and deterministic

## Interpreting Results

**If classes are identical**: The optimization may not affect these particular macro patterns, or the macros don't actually emit syntax-quoted nil in their expansion.

**If classes differ**: The optimization changes compiled bytecode. Examine the diff files to understand:
- Which methods changed
- What instructions were added/removed
- How much bytecode size changed
